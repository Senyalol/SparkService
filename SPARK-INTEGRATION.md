# Интеграция Apache Spark (Structured Streaming)

Приложение читает транзакции из Kafka (`user-transactions`), детектирует аномалии и считает RFM-сегменты, пишет результаты в отдельные топики.

## Этап 1: Детекция аномалий

- **Состояние по пользователю:** скользящее окно 5 минут — хранятся суммы транзакций для расчёта `avg_check_5min`.
- **Правило:** если пришла транзакция типа **Credit** (снятие) и её сумма **≥ 3 × avg_check_5min**, транзакция считается аномальной.
- **Сброс состояния:** если от пользователя нет активности 5 минут, состояние сбрасывается (таймаут Spark state).
- **Выход:** топик **`alerts`** (JSON: user_id, event_time, type, sum, avg_check_5min, message).

## Этап 2: RFM-сегментация

- **R (Recency):** время с последней транзакции до текущего момента (минуты).
- **F (Frequency):** число транзакций за последние 24 часа.
- **M (Monetary):** сумма всех **Deposit** за последние 24 часа.

**Сегменты:**

| Сегмент      | Условие                          |
|-------------|-----------------------------------|
| VIP         | M > 10000 и F > 5                |
| Активный    | M > 1000 и F > 1                 |
| Новичок     | первая транзакция &lt; 1 часа назад |
| Спящий      | R > 30 мин и не Новичок          |
| Стандартный| остальные                         |

- **Выход:** топик **`user-segments`** (JSON: user_id, segment, r_minutes, f, m, updated_at).

## Переменные окружения

| Переменная               | По умолчанию        | Описание                    |
|--------------------------|---------------------|-----------------------------|
| KAFKA_BOOTSTRAP_SERVERS  | kafka:9092          | Брокеры Kafka               |
| KAFKA_TOPIC              | user-transactions   | Входной топик               |
| KAFKA_ALERTS_TOPIC       | alerts              | Топик для аномалий          |
| KAFKA_SEGMENTS_TOPIC      | user-segments       | Топик для RFM-сегментов     |

## Checkpoint

Spark сохраняет состояние стримов в:

- `/tmp/spark-alerts-checkpoint`
- `/tmp/spark-segments-checkpoint`

В Docker убедись, что `/tmp` доступен для записи или смени пути через конфиг Spark.

## Запуск

- **Локально:** `mvn compile exec:java -Dexec.mainClass="Main"`
- **Docker:** образ собирается как раньше; при запуске в одной сети с Kafka топики `alerts` и `user-segments` создаются при первой записи.

Типы транзакций в JSON от Producer должны совпадать с логикой: **Credit** (снятие) для аномалий, **Deposit** для расчёта M.
